双指针问题

环形链表

给定一个链表，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true 。 否则，返回 false

链接：https://leetcode-cn.com/leetbook/read/linked-list/jbex5/

思路：定义两个指针，一个快指针，一个慢指针，快指针一下走两步，满指针一下子走一步，加入能够让两个指针相遇，则能证明，链表中存在换环。

环形链表plus

给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。注意，pos 仅仅是用于标识环的情况，并不会作为参数传递到函数中。

说明：不允许修改给定的链表。

作者：力扣 (LeetCode)
链接：https://leetcode-cn.com/leetbook/read/linked-list/jjhf6/
来源：力扣（LeetCode）
著作权归作者所有。商业转载请联系作者获得授权，非商业转载请注明出处。

思路：首先，快慢指针定义和上次一样，然后你设慢指针走了n，则快指针走了2n步，设进环之前满指针走了x步，则进换环之后走了n-x步，那么把快指针指向头，则等待n-x步之后，快指针与慢指针相交的地点，就是进环的第一个结点

相交链表

走到尽头见不到你，于是走过你来时的路，等到相遇时才发现，你也走过我来时的路。

思路：走一样的路程，一样的速度肯定会相遇的

删除链表

给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

思路：你让它们两个相差n个位置，则当快的到末尾是，慢的恰好就在倒数第n个结点

约瑟夫问题

约瑟夫问题：n个人围成一圈，初始编号从1~n排列，从约定编号为x的人开始报数，数到第m个人出圈，接着又从1开始报数，报到第m个数的人又退出圈，以此类推，最后圈内只剩下一个人，这个人就是赢家，求出赢家的编号

思路：三种思路，递归，环形链表，直接遍历；递归：主要是那求那个变化前与变化后的 假设 old 为删除之前的节点编号， new 为删除了一个节点之后的编号，则 old 与 new 之间的关系为 old = (new + m - 1) % n + 1。注：编号从一开始，所以要加一；环形链表就是比直接遍历降低了事件复杂度
